# IMS 注册状态显示修复

日期：2026-06-20

## 目标
- 修复 IMS 状态读取与 CarrierConfig 读取并发时结果类型串错的问题。
- 区分 IMS 已注册、未注册、读取失败和注册中，避免把所有非成功状态都显示成普通关闭。
- 点击触发 IMS 注册后保留注册中状态，并在短时间内自动重试读取。
- 注册仍失败时给出简短提示：建议到信号更稳定的位置稍后再试。

## 非目标
- 不修改运营商配置规则。
- 不自动判断或修改 IMS APN。
- 不把数据网络状态当成 IMS 注册状态。

## 行为变化
- 特权 instrumentation 调用串行执行，减少多个读取任务同时返回导致的结果错配。
- `IMS 注册状态` 在触发注册期间右侧显示“注册中”，并暂时禁用点击。
- 注册触发后不再立刻只读一次状态，而是延迟重试几次后再给最终结果。
- 读取失败时文案显示为未知，不再和明确未注册完全混在一起。

## 影响范围
- 首页 IMS 功能卡片。
- IMS 状态读取、CarrierConfig 读取和 IMS 重启后的状态同步。
- QS 图块继续复用同一套 IMS 状态读取能力。

## 验证方式
- 运行单元测试和 Release 构建。
- 安装到手机后验证：点击 IMS 注册时显示“注册中”，成功显示开启，失败显示简短失败提示。
- 查看日志确认 IMS 状态读取会输出 subId 和返回值，且不再出现 `Key result expected Bundle but value was a java.lang.Boolean`。
